# Unity Enemy AI & Weapon System

## 项目简介

这是一个基于 Unity 实现的 2D Top-Down 战斗系统项目，核心内容包括敌人 AI、武器瞄准与射击、换弹动画、战斗特效、音效管理以及角色动画控制。

项目重点并不只是“完成一个可运行的小游戏”，而是围绕战斗体验与系统设计，构建了一套较完整的敌人行为逻辑与武器表现系统，适合作为 Unity 游戏开发方向的作品集项目进行展示。

## 核心功能

### 1. 敌人 AI 状态控制
项目中的敌人具备完整的行为状态切换逻辑，包括：

- 巡逻（Roaming）
- 发现目标后追击（ChaseTarget）
- 进入攻击范围后射击（ShootingTarget）
- 丢失目标后返回初始位置（GoingBackToStart）

通过状态切换实现基础战斗 AI，使敌人具有一定的交互性和追击感，而不是简单地朝玩家直线移动。

### 2. 武器瞄准与射击系统
实现了角色武器瞄准、射击和攻击方向切换：

- 根据目标位置动态调整瞄准方向
- 根据朝向切换左右武器动画
- 射击时播放独立射击动画
- 支持攻击完成后的动画回切

这部分的设计使角色战斗表现更加自然，也提升了项目在动作反馈上的完整度。

### 3. 换弹系统
在基础射击逻辑之外，补充了独立的换弹动画控制：

- 根据目标方向播放对应换弹动作
- 换弹过程中锁定对应动画状态
- 换弹结束后恢复正常瞄准状态

这让武器系统不只是“发射子弹”，而是具备更完整的战斗流程。

### 4. 战斗特效系统
为了增强战斗表现，项目实现了多种即时反馈特效：

- 枪口闪光
- 子弹拖尾
- 血液飞溅
- 尘土粒子

这些特效不是简单堆叠，而是结合统一的 Mesh 更新逻辑进行管理，减少频繁实例化带来的性能浪费。

### 5. 音效管理系统
项目实现了统一的声音管理模块：

- 管理多种战斗音效
- 支持同类音效随机播放，避免重复感
- 使用对象池复用 AudioSource
- 支持 2D / 3D 音效播放控制

这样既保证了战斗反馈的丰富度，也避免了频繁创建音频对象带来的性能问题。

### 6. 角色动画控制
项目对角色动画进行了封装设计，包括：

- Idle / Walk 动画切换
- 基于移动方向的动画播放
- 武器动画与身体部位动画组合控制
- 不同朝向下的动作适配

这部分体现了角色表现层与逻辑层之间的解耦思路。

## 项目亮点

### 亮点一：敌人 AI 逻辑完整
相比简单的“发现玩家就追击”，本项目实现了巡逻、追击、射击、返回四类状态，使敌人行为更接近真实战斗场景。

### 亮点二：战斗系统结构清晰
将瞄准、射击、换弹、动画切换分别拆分管理，避免所有逻辑混杂在单一脚本中，便于维护和扩展。

### 亮点三：有性能优化意识
粒子与特效没有全部采用高频实例化方式，而是通过统一 Mesh 管理逻辑进行批量更新，体现了对性能的关注。

### 亮点四：项目可扩展性强
当前系统已经具备进一步扩展的基础，例如：
- 增加更多武器类型
- 加入敌人近战 / 远程区分
- 增加伤害数值系统
- 接入背包、掉落、关卡机制
- 拓展更复杂的行为树 / 状态机 AI
